Text color

With CSS, a color is most often specified by:

  • A color name like "red"
  • A HEX value like "#ff0000"
  • An rgb value like "rgb(255, 0, 0)"
p {
  color: red;
  color: #ff0000;
  color: rgb(255, 0, 0);
}

For W3C compliant CSS: If you define the color property, you must also define the background-color.

Text decoration

text-decoration allows you to add or remove decorations from text.

The value text-decoration: none; is often used to remove underlines from links.

h1 {
  text-decoration: overline;
}

h2 {
  text-decoration: line-through;
}

h3 {
  text-decoration: underline;
}

h3 {
  text-decoration: none;
}

Transform text to uppercase/lowercase/capitalize

text-transform allows you to turn everything into uppercase or lowercase letters, or capitalize the first letter of each word.

p.uppercase {
  text-transform: uppercase;
}

p.lowercase {
  text-transform: lowercase;
}

p.capitalize {
  text-transform: capitalize;
}

p.capitalize {
  text-transform: none;
}

Text direction

direction is used to specify which direction the text grows from.

/* text position to right and moves to left while typing */
div {
  direction: rtl;
}

/* text position to left and moves to right while typing */
div {
  direction: ltr;
}

text-shadow

text-shadow adds a shadow to the text in different directions. The following example specifies the position of the horizontal shadow (3px), the position of the vertical shadow (2px) and the color of the shadow (red):

white-space

white-space specifies how white-space is handled inside an element. Value are:

  • normal - This is default
  • nowrap - Text will never wrap to the next line. The text continues on the same line until a <br> tag is encountered.
  • pre - Text will never wrap to the next line. Acts like the <pre> tag in HTML. Text will only wrap if the HTML is formatted that way, or if you use a <br> tag. Text will also indent according to how the HTML is formatted.
  • pre-line - Same as <pre> in that text will wrap how HTML is formatted, or if you use a <br> tag. But it's different in that text will still wrap when necessary, for example, if the browser window can't view it all, it will automatically wrap. Also, HTML formatting is ignored for indented text. Only line breaks are recognized in HTML.
  • pre-wrap - This is the same as pre-line, except that indented text is recognized, just like 'pre'. The difference from 'pre' is that the text eventually wraps.

Example: nowrap

Ignore the scrollbar. Text will be cut off and never move down to the next line.

p {
  white-space: nowrap;
}
This is a very long line that by default, should break to the next line of text so the reader can view all text.
